BMS.OTProto.load_labware
Example
¶
Using the native Opentrons API, the way in which labware is loaded can change depending on whether the labware is custom or default, and whether the protocol is being executed via the app, executed via a jupyter notebook, or if it is being simulated. The
OTProto.load_labware
function can load any labware in any environment.

Best Practices
¶
It is recommended that a directory/folder is created for custom labware files on any computer that will be used for developing/simulating Opentrons protocols. This directory can then always be provided where appropriate. When the protocol is loaded to the Opentrons via the app, the directory will then be ignored so that custom labware can be loaded through the usual way.

¶
Any labware being loaded must be reference by its

https://labware.opentrons.com/
When loading to the deck, the
parent

Loading Labware to a Hardware Module
¶
Labware can be loaded to a hardware module by using the hardware module's object as the value for the
parent
argument. In this case, any deck position will be ignored as the labware will be loaded to wherever the hardware module is loaded

Loading Custom Labware
¶
To load custom labware, the same as above can be followed, except a custom labware directory containing the
.json
file for the labware must be included.
To create definitions for your own custom labware, see here:
https://labware.opentrons.com/create
In [16]:
Copied!
custom_labware_dir
=
"../../../For docs/custom_labware/"
custom_labware_dir = "../../../For docs/custom_labware/"

API name
is the name of the file (without the extension)
